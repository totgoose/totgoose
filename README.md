# Тот Ещё Гусь — Сайт гастрокафе

Статический сайт для гастрокафе "Тот Ещё Гусь" в Москве. Разработан для размещения на GitHub Pages с максимальной SEO-оптимизацией.

## 🚀 Быстрый старт

### 1. Создайте репозиторий на GitHub

1. Зайдите на [github.com](https://github.com)
2. Создайте новый репозиторий с именем `totgoose` (или любым другим)
3. Сделайте его публичным (Public)

### 2. Загрузите файлы

**Вариант A: Через веб-интерфейс**
1. Откройте ваш репозиторий
2. Нажмите "Add file" → "Upload files"
3. Перетащите все файлы из папки `totgoose-site`
4. Нажмите "Commit changes"

**Вариант B: Через Git**
```bash
cd totgoose-site
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/ВАШ_НИК/totgoose.git
git push -u origin main
```

### 3. Включите GitHub Pages

1. В репозитории перейдите в **Settings** → **Pages** [^1^]
2. В разделе "Source" выберите:
   - Branch: `main` (или `master`)
   - Folder: `/(root)`
3. Нажмите **Save**

### 4. Проверьте сайт

Через 2-5 минут сайт будет доступен по адресу:
```
https://ВАШ_НИК.github.io/totgoose
```

## ✨ Особенности

### Дизайн
- **Адаптивный дизайн** — работает на мобильных и десктопах
- **Фирменный стиль** — цвета из вашего дизайна (кремовый фон, красные акценты)
- **Анимации** — плавные переходы, hover-эффекты, fade-in при загрузке
- **Accessibility** — поддержка keyboard navigation, ARIA-метки

### Функциональность
- **10 категорий меню** — 68 блюд с подробными описаниями
- **Отдельные страницы блюд** — каждое блюдо имеет свою URL для SEO
- **Быстрая навигация** — мгновенный переход между категориями
- **История и факты** — каждое блюдо содержит историю, интересные факты, сочетание с вином

### SEO-оптимизация [^2^]
- **Sitemap.xml** — карта сайта с 80+ URL для индексации
- **Robots.txt** — разрешение на индексацию всех страниц
- **Meta-теги** — уникальные title и description для каждой страницы
- **Open Graph** — превью при шеринге в соцсетях
- **Структурированные URL** — ЧПУ для каждого блюда
- **Semantic HTML** — правильная структура заголовков (H1-H6)

## 🔧 Настройка собственного домена

Если хотите использовать свой домен (например, `totgoose.ru`):

### 1. Настройте DNS [^1^]

У вашего DNS-провайдера создайте записи:

**A-записи** (для корневого домена):
```
185.199.108.153
185.199.109.153
185.199.110.153
185.199.111.153
```

**AAAA-записи** (для IPv6):
```
2606:50c0:8000::153
2606:50c0:8001::153
2606:50c0:8002::153
2606:50c0:8003::153
```

**CNAME** (для www): `www` → `ВАШ_НИК.github.io`

### 2. Настройте GitHub Pages

1. В Settings → Pages введите ваш домен в поле "Custom domain"
2. Дождитесь проверки DNS (может занять до 24 часов)
3. Поставьте галочку "Enforce HTTPS" для безопасного соединения

### 3. Обновите sitemap.xml

Замените `https://totgoose.github.io/` на ваш домен в файле `sitemap.xml`.

## 📝 Обновление меню

Данные меню хранятся в `menu-data.js`. Для обновления:

1. Отредактируйте файл `menu-data.js`
2. Для добавления нового блюда используйте структуру:
```javascript
{
  "name": "Название блюда",
  "slug": "url-slug",
  "description": "Описание",
  "price": "1200",
  "calories": "~350 ккал",
  "time": "45 мин",
  "origin": "Италия",
  "history": "История блюда...",
  "facts": "• Факт 1\n• Факт 2",
  "wine": "Вино",
  "image": "https://...",
  "url": "dishes/url-slug.html"
}
```
3. Загрузите новую версию в репозиторий
4. Изменения появятся через 1-2 минуты

## 🐛 Устранение неполадок

### Сайт не загружается
- Проверьте, что репозиторий публичный
- Убедитесь, что файлы загружены в корень (не в подпапку)
- Подождите 5-10 минут после загрузки

### Изображения не отображаются
- Проверьте пути к изображениям (относительные пути: `goose_c.png`, `dishes/...`)
- Убедитесь, что файлы изображений загружены

### DNS не работает
- DNS-изменения могут занимать до 24 часов
- Проверьте записи через `dig` или онлайн-инструменты
- Убедитесь, что нет конфликтующих записей
